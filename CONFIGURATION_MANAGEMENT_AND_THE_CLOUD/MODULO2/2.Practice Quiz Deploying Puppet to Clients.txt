1)When defining nodes, how do we identify a specific node that we want to set rules for?
Resp)By specifying the node's Fully Qualified Domain Names (FQDNs)

2)When a Puppet agent evaluates the state of each component in the manifest, it uses gathered facts about the system to decide which rules to apply.
 What tool can these facts be "plugged into" in order to simplify management of the content of our Puppet configuration files?
Resp)Templates

3)What is the first thing that happens after a node connects to the Puppet master for the first time?
Resp)The node requests a certificate.

4)What does FQDN stand for, and what is it?
Resp)Fully Qualified Domain Name, which is the full address for a node

5)What type of cryptographic security framework does Puppet use to authenticate individual nodes?
Resp)Public Key Infrastructure (PKI)
